Palestrantes

Georges Boris

Tasking

Georges Boris

Fundador da Tasking, uma startup de gestão de produtividade para equipes criativas, mostrará com uma equipe de apenas dois desenvolvedores construíram um SaaS utilizando React, Redux e Firebase.

Mostrarão para vocês como uma equipe tão enxuta pode fazer rapidamente uma aplicação limpa, performática e facilmente escalável, graças a separação completa de preocupações através de uma interface funcional (react), middleware funcional (redux) e backend reativo (firebase + firebase functions).

Filipe Monteiro

Nutripad

Filipe Monteiro

Desenvolver aplicativos móveis nativos não deveriam ser tão complicados, mas infelizmente é. Porém o React Native tem tornado esse processo muito prazeroso e fácil de trabalhar, entenda como desenvolvemos um app novo em apenas 16 dias.

Filipe compartilhou no Medium essa experiência e vai demonstrar em detalhes pra vocês nessa conferência.

Wagner Horta Neto

GreenMile

Wagner Horta Neto

Experimentaram ao longo dos últimos 10 anos todo tipo de soluções Mobile desde o JME a híbridos.

Tem que garantirem a sincronia de um Backend que recebe mais de 700 milhões de requisições por mês com milhares de devices Android, IOS e Windows Phone e centenas de milhares de clientes monitorando rotas com todos os navegadores modernos.

Nesta apresentação apresentarão as motivações que fizeram escolher o stack de React com Cordova.

Cláudio Fortier

PagueMenos

Cláudio Fortier

Coordenador de Inovação Tecnológica e Arquiteto da rede de Farmácias Pague Menos, apresentarão como foi o processo de validar uma alternativa para o Asp Net MVC.

Buscando uma solução para desenvolvimento de front end web desacoplado do backend, apostaram na interface reativa com ReactJS e colocaram seu primeiro aplicativo em produção 78 dias depois de iniciar os estudos.

As iniciativas atualmente em funcionamento incluem integração com Backends em .Net Core e construção de um cockpit para o Microsoft Visual Studio Team Services, onde os fontes em ReactJS fazem parte das rotinas de build automático no modelo de DEVOPS.
